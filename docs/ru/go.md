# Go

::: info Информация
Последняя стабильная версия `uuid` — **v1.26.11**.
:::

## Get Started
```bash
go get github.com/mikhaildadaev/uuid
```

## Run Test 
```bash
go test ./...
go test -bench=. ./...
go test -cover ./...
go test -race ./...
```

## Key Features
- **Унифицированный API** — Единый API для логов, метрик и трейсов.
- **Извлечение контекста** — Автоматическое извлечение `node_id`, `trace_id` и т.д. из `context.Context`.
- **Цветной вывод** — `Dark` и `Light` темы с автоопределением для TEXT формата.
- **16 типов полей** — `Bool`, `Bools`, `Duration`, `Durations`, `Error`, `Errors`, `Float64`, `Floats64`, `Int`, `Ints`, `Int64`, `Ints64`, `String`, `Strings`, `Time`, `Times`.
- **Запись в файл** — Неблокирующая атомарная ротация со сжатием `gzip`.
- **Запись по сети** — `Batching`, `Circuit Breaker`, `Deduplication`, `Retry`, `Sampling`.
- **8 интеграций** — `Discord`, `Kafka`, `Loki`, `Prometheus`, `Slack`, `Telegram`, `Tempo`, `WeChat`.

## Limits
- **Асинхронный буфер**: при переполнении запись выполняется синхронно (без блокировки)
- **Информация о вызывающем коде**: только для `LevelDebug` (оптимизация производительности)
- **Точность времени**: микросекунды (6 знаков) — достаточно для 99% случаев, снижает аллокации
- **Кеш дедупликации**: только в памяти, периодически очищается (без сохранения между перезапусками)
- **Circuit Breaker**: сбрасывается при перезапуске приложения (без хранения состояния)
- **Ротация файлов**: проверка размера при каждой записи; ротация при первом превышении лимита
- **HTTP-батчинг**: сообщения могут быть потеряны при аварийном завершении до сброса
- **Kafka**: используется REST Proxy API (не нативный протокол) — требуется Confluent REST Proxy
- **Loki**: используется HTTP API (`/loki/api/v1/push`) — метки должны быть настроены заранее
- **Извлечение контекста**: работает только со значениями, сохранёнными через `context.WithValue()`
- **Нулевые зависимости**: по дизайну; никаких внешних библиотек для функций вроде нативного протокола Kafka
