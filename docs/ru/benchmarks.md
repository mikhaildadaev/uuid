# Бенчмарки

::: info Информация
Лучший способ сравнить библиотеки - выполнить тесты в **вашей собственной среде** с **вашей собственной рабочей нагрузкой**. Каждый проект предъявляет уникальные требования — задержка, пропускная способность, объем памяти, сложность интеграции — и ни один тест не может учесть их все.

Рекомендую вам протестировать `uuid` наряду с другими библиотеками и выбрать инструмент, который наилучшим образом соответствует вашим потребностям.
:::

## Core Performance
Измеряются только накладные расходы на генерацию и сериализацию. Бенчмарки пишут в `io.Discard` — операции ввода-вывода исключены.

::: tip Примечание
Все бенчмарки измеряют чистые накладные расходы на генерацию и парсинг. Бенчмарки, отмеченные как zero-allocation (`0 B/op`, `0 allocs/op`), не выполняют выделений памяти в куче на критическом пути исполнения — незначительные артефакты в однопоточных V3/V5 исходят от самого тестового фреймворка Go, а не от генерации UUID. Реальная производительность зависит от процессора, задержек памяти и конкурентности.

*Протестировано на Intel Core i9-9880H (2.30 ГГц).*
:::
