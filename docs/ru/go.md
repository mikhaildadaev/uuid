## Key Features
- **UUIDv1..v8 и Null** — Генерация всех версий UUID и SQL-совместимого null UUID;
- **Парсинг и Валидация** — Парсинг и проверка стандартных и null UUID строк;
- **Извлечение Метаданных** — Извлечение временной метки, последовательности, узла, POSIX, варианта и версии;
- **Сериализация** — Полная поддержка Binary, JSON и Text marshal/unmarshal;
- **SQL Интеграция** — Нативная поддержка database/sql driver с интерфейсами Scan/Value;
- **Zero-Allocation** — Быстрые, без выделений памяти, основные процедуры на всех критических путях.

## Limits
- **V2**: `posix` должен быть `0..255`;
- **V3/V5**: `name` должен быть `1..512` символов (длина в байтах);
- **V8**: `node` должен быть `0..16383`;

::: tip Примечание 
Строки `name` длиной 0 байт или более 512 байт возвращают null UUID для предотвращения выделения памяти.
:::
